Shared Backend

parcel-api

parcel-api was the shared backend platform behind both the internal iCargo operations UI and the iCargo Customer portal.

Visual Overview

Reference visual summarizing the shared auth context and cross-product services behind parcel-api.

Why It Matters

This is the central business layer story. It shows one backend serving multiple product surfaces instead of a small helper API.

Tech Stack

  • PHP backend platform
  • Redis-backed business workflows
  • token-based API access and shared context
  • order-ingestion and logistics APIs

What I Worked On

  • shipment booking, pricing, route, and webhook flows
  • shared APIs used by both internal and customer-facing products
  • customer settings and authenticated request context
  • external order-ingestion workflows from commerce platforms
